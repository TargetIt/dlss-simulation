# 管理原则与方法

## 工作哲学

1. **先理解，再动手**：每个模块先研究原理（读论文/博客/源码），再写模拟代码
2. **MVP 优先**：先做最简单的能跑的版本，再逐步完善
3. **素材卡片化**：研究过程中收集的参考资料用模板记录（复用 tech-popularization 项目的 素材卡片模板.md）
4. **正确性 > 性能**：模拟结果可以不快，但不能原理性错误

## 单次工作流程

```
1. 读 PM/05-需求记录.md    → 确认今天干什么
2. 读 PM/02-项目状态.md     → 了解当前进度
3. 执行任务                → 研究 / 编码 / 跑实验
4. 更新 PM 文件             → 实时更新需求和状态
5. 记录决策到 04-决策记录   → 如有重要决定
```

## 代码规范

- Python 代码放在 `src/` 下按模块组织
- 实验脚本放在 `experiments/` 下
- 测试数据放在 `data/` 下（不提交 git）
- 所有超参数在代码中显式声明，不硬编码
